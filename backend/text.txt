**AWS S3 / CLOUDFLARE R2**

**Option A — AWS S3:**
1. Go to [aws.amazon.com](https://aws.amazon.com) → IAM → Create a user
2. Attach the `AmazonS3FullAccess` policy (or a scoped policy for just your buckets)
3. Create access keys → copy `Access Key ID` and `Secret Access Key`
4. Go to S3 → Create two buckets:
   - `netprime-raw-uploads` (private)
   - `netprime-hls-processed` (private, CDN will serve from here)
5. Set your region (e.g. `us-east-2`)
6. Leave `S3_ENDPOINT` blank

**Option B — Cloudflare R2 (recommended — no egress fees):**
1. Go to [dash.cloudflare.com](https://dash.cloudflare.com) → R2
2. Create two buckets: `netprime-raw-uploads` and `netprime-hls-processed`
3. Go to **R2 → Manage R2 API Tokens** → Create token with Object Read & Write
4. Copy the Access Key ID and Secret Access Key
5. Your endpoint is: `https://<your_account_id>.r2.cloudflarestorage.com`
6. Set `AWS_REGION=auto`

---

**CDN — skip for local dev**

Leave all CDN variables blank while testing locally. Your stream service will fall back to direct S3 URLs which is fine for Postman testing. You only need this configured for production.

When you're ready for production:
- **CloudFront**: AWS Console → CloudFront → Create distribution pointing to your HLS bucket. Generate a key pair under **CloudFront → Public keys**.
- **Cloudflare**: Add your domain to Cloudflare, point it at your R2 bucket, generate an HMAC secret with `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`.

---

**EMAIL (SMTP)**

**Option A — Gmail (quickest for dev):**
1. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
2. Select **Mail** → your device → Generate
3. Copy the 16-character app password (not your normal Gmail password)
```
SMTP_USER=youremail@gmail.com
SMTP_PASS=xxxx xxxx xxxx xxxx    ← the 16-char app password
```

**Option B — Resend (cleanest free tier for dev):**
1. Go to [resend.com](https://resend.com) → get an API key
```
SMTP_HOST=smtp.resend.com
SMTP_PORT=465
SMTP_USER=resend
SMTP_PASS=re_xxxxxxxxxxxx    ← your Resend API key
```

**Dev tip:** Leave `SMTP_USER` blank entirely and emails just get logged to the console — useful while testing auth before you have SMTP set up.
